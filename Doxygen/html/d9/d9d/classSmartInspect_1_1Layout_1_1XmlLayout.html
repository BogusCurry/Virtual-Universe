Detailed Description

Layout that formats the log events as XML elements.

The output of the XmlLayout consists of a series of SmartInspect:event elements. It does not output a complete well-formed XML file. The output is designed to be included as an external entity in a separate file to form a correct XML file.

For example, if abc is the name of the file where the XmlLayout output goes, then a well-formed XML file would be:

&lt;?xml version="1.0" ?&gt;
&lt;!DOCTYPE SmartInspect:events SYSTEM "SmartInspect-events.dtd" [&lt;!ENTITY data SYSTEM "abc"&gt;]&gt;
&lt;SmartInspect:events version="1.2" xmlns:SmartInspect="http://logging.apache.org/SmartInspect/schemas/SmartInspect-events-1.2&gt;
&amp;data;
&lt;/SmartInspect:events&gt;

This approach enforces the independence of the XmlLayout and the appender where it is embedded.

The version attribute helps components to correctly interpret output generated by XmlLayout. The value of this attribute should be "1.2" for release 1.2 and later.

Alternatively the Header and Footer properties can be configured to output the correct XML header, open tag and close tag. When setting the Header and Footer properties it is essential that the underlying data store not be appendable otherwise the data will become invalid XML.

Constructs an XmlLayout.

The LocationInfo option takes a boolean value. By default, it is set to false which means there will be no location information output by this layout. If the the option is set to true, then the file name and line number of the statement at the origin of the log statement will be output.

If you are embedding this layout within an SmtpAppender then make sure to set the LocationInfo option of that appender as well.

Property Documentation

bool SmartInspect.Layout.XmlLayout.Base64EncodeMessage
getset

Set whether or not to base64 encode the message.

By default the log message will be written as text to the xml output. This can cause problems when the message contains binary data. By setting this to true the contents of the message will be base64 encoded. If this is set then invalid character replacement (see XmlLayoutBase.InvalidCharReplacement) will not be performed on the log message.

bool SmartInspect.Layout.XmlLayout.Base64EncodeProperties
getset

Set whether or not to base64 encode the property values.

By default the properties will be written as text to the xml output. This can cause problems when one or more properties contain binary data. By setting this to true the values of the properties will be base64 encoded. If this is set then invalid character replacement (see XmlLayoutBase.InvalidCharReplacement) will not be performed on the property values.

string SmartInspect.Layout.XmlLayout.Prefix
getset

The prefix to use for all element names

The default prefix is SmartInspect. Set this property to change the prefix. If the prefix is set to an empty string then no prefix will be written.
