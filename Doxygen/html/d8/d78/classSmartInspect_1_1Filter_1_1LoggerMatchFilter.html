Detailed Description

Simple filter to match a string in the event's logger name.

The works very similar to the LevelMatchFilter. It admits two options LoggerToMatch and AcceptOnMatch. If the LoggingEvent.LoggerName of the LoggingEvent starts with the value of the LoggerToMatch option, then the Decide method returns FilterDecision.Accept in case the AcceptOnMatch option value is set to true, if it is false then FilterDecision.Deny is returned.

The rendered message is matched against the LoggerToMatch. If the LoggerToMatch equals the beginning of the incoming LoggingEvent.LoggerName (M:String.StartsWith(string)) then a match will have occurred. If no match occurs this function will return FilterDecision.Neutral allowing other filters to check the event. If a match occurs then the value of AcceptOnMatch is checked. If it is true then FilterDecision.Accept is returned otherwise FilterDecision.Deny is returned.

Property Documentation

bool SmartInspect.Filter.LoggerMatchFilter.AcceptOnMatch
getset

FilterDecision.Accept when matching LoggerToMatch

The AcceptOnMatch property is a flag that determines the behavior when a matching Level is found. If the flag is set to true then the filter will FilterDecision.Accept the logging event, otherwise it will FilterDecision.Deny the event.

The default is true i.e. to FilterDecision.Accept the event.

string SmartInspect.Filter.LoggerMatchFilter.LoggerToMatch
getset

The LoggingEvent.LoggerName that the filter will match

This filter will attempt to match this value against logger name in the following way. The match will be done against the beginning of the logger name (using M:String.StartsWith(string)). The match is case sensitive. If a match is found then the result depends on the value of AcceptOnMatch.
