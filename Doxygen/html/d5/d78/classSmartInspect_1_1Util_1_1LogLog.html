Detailed Description

Outputs log statements from within the SmartInspect assembly.

SmartInspect components cannot make SmartInspect logging calls. However, it is sometimes useful for the user to learn about what SmartInspect is doing.

All SmartInspect internal debug calls go to the standard output stream whereas internal error messages are sent to the standard error output stream.

bool SmartInspect.Util.LogLog.InternalDebugging
staticgetset

Gets or sets a value indicating whether SmartInspect internal logging is enabled or disabled.

true if SmartInspect internal logging is enabled, otherwise false.

When set to true, internal debug level logging will be displayed.

This value can be set by setting the application setting SmartInspect.Internal.Debug in the application configuration file.

The default value is false, i.e. debugging is disabled.

The following example enables internal debugging using the application configuration file :

<configuration>
<appSettings>
<add key="SmartInspect.Internal.Debug" value="true" />
</appSettings>
</configuration>

bool SmartInspect.Util.LogLog.QuietMode
staticgetset

Gets or sets a value indicating whether SmartInspect should generate no output from internal logging, not even for errors.

true if SmartInspect should generate no output at all from internal logging, otherwise false.

When set to true will cause internal logging at all levels to be suppressed. This means that no warning or error reports will be logged. This option overrides the InternalDebugging setting and disables all debug also.

This value can be set by setting the application setting SmartInspect.Internal.Quiet in the application configuration file.

The default value is false, i.e. internal logging is not disabled.

The following example disables internal logging using the application configuration file :

<configuration>
<appSettings>
<add key="SmartInspect.Internal.Quiet" value="true" />
</appSettings>
</configuration>
